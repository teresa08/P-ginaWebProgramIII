Spring Framework

El framework más popular para el desarrollo de aplicaciones empresariales en Java.

¿Qué es Spring Framework?

Spring Framework es un framework de código abierto para la plataforma Java que proporciona infraestructura integral para desarrollar aplicaciones robustas, escalables y mantenibles. Fue creado por Rod Johnson en 2002 y desde entonces se ha convertido en el estándar de facto para aplicaciones empresariales en Java.

Su principal objetivo es simplificar el desarrollo de aplicaciones Java mediante la inversión de control (IoC) y la inyección de dependencias (DI), reduciendo el acoplamiento entre componentes.

Funcionalidades clave

  • Inversión de Control (IoC): Spring gestiona el ciclo de vida y la configuración de los objetos (llamados "beans").
  • Inyección de Dependencias (DI): Las dependencias se inyectan automáticamente, no se crean manualmente con new.
  • Programación orientada a aspectos (AOP): Permite separar lógica transversal (logging, seguridad, transacciones) del código de negocio.
  • Soporte para transacciones declarativas: Gestión de transacciones sin código repetitivo.
  • Integración con bases de datos: Plantillas JDBC, soporte para JPA, Hibernate, y más.
  • Pruebas unitarias y de integración: Soporte nativo para pruebas con JUnit y Mockito.
  • Flexibilidad: No impone una arquitectura específica; se adapta a tus necesidades.

Módulos principales de Spring Framework

Spring Core

Contiene las funcionalidades básicas: IoC y DI.

Spring Context

Construye sobre Core y agrega soporte para mensajes, eventos y recursos.

Spring AOP

Implementa programación orientada a aspectos.

Spring JDBC

Simplifica el acceso a bases de datos con plantillas y manejo de excepciones.

Spring ORM

Integración con JPA, Hibernate, JDO, etc.

Spring Web

Soporte para aplicaciones web (base para Spring MVC).

Spring Test

Herramientas para pruebas en contexto de Spring.

Ejemplo básico: Configuración con anotaciones

Definimos un servicio y lo inyectamos en un controlador usando Spring.

Servicio

@Service
public class SaludoService {
    public String generarSaludo(String nombre) {
        return "¡Hola, " + nombre + "!";
    }
}

Controlador (Spring MVC)

@Controller
public class SaludoController {
    
    @Autowired
    private SaludoService saludoService;

    @GetMapping("/saludar")
    @ResponseBody
    public String saludar() {
        return saludoService.generarSaludo("María");
    }
}

Spring se encarga de crear la instancia de SaludoService e inyectarla en el controlador.

Ventajas de usar Spring

  • ✅ Bajo acoplamiento: los componentes son independientes y reemplazables.
  • ✅ Reusabilidad: los beans se pueden reutilizar en distintos contextos.
  • ✅ Productividad: reduce el código repetitivo (boilerplate).
  • ✅ Comunidad y ecosistema: millones de desarrolladores, documentación extensa y soporte.
  • ✅ Integración: se conecta fácilmente con JPA, Hibernate, Kafka, Redis, etc.
Nota: Spring Framework es la base. Spring Boot (creado posteriormente) simplifica aún más la configuración con autoconfiguración y starters.

¿Cuándo usar Spring Framework?

  • ✅ Aplicaciones empresariales complejas.
  • ✅ Sistemas que requieren gestión de transacciones.
  • ✅ Proyectos que necesitan escalabilidad y mantenibilidad a largo plazo.
  • ✅ Integración con múltiples tecnologías (bases de datos, colas, servicios web).
