Guía Completa sobre XML

XML (eXtensible Markup Language) es un lenguaje de marcado diseñado para almacenar, transportar y describir datos de forma estructurada, legible y compatible entre diferentes plataformas y lenguajes de programación.

¿Para qué sirve XML?

  • Intercambio de datos entre aplicaciones o sistemas (especialmente en servicios web SOAP).
  • Configuración de software (archivos como web.config, pom.xml, etc.).
  • Almacenamiento de datos estructurados cuando no se usa una base de datos.
  • Formato base en estándares como RSS, SVG, XHTML, Office Open XML (.docx, .xlsx).

Características clave

  • Autodescriptivo: las etiquetas definen el significado del contenido.
  • Extensible: puedes crear tus propias etiquetas según tus necesidades.
  • Independiente: funciona en cualquier sistema operativo o lenguaje.
  • Validable: se puede verificar con DTD o XML Schema (XSD).

Sintaxis básica y reglas esenciales

  • Toda etiqueta de apertura debe tener su cierre: ....
  • Las etiquetas distinguen mayúsculas y minúsculas: ≠ .
  • Debe haber un solo elemento raíz que contenga a todos los demás.
  • Los atributos deben ir entre comillas: moneda="USD".
  • Los caracteres especiales deben escaparse (&lt;, &amp;, etc.).

Ejemplo práctico: Lista de invitación familiar

En este ejemplo, <ListaInvitación> es el elemento raíz. Contiene una <familia>, que a su vez incluye una <tía> con dos nombres: 

<?xml version="1.0" encoding="UTF-8"?>
<ListaInvitación>
  <familia>
    <tía>
      <nombre>Cristina</nombre>
      <nombre>Estefanía</nombre>
    </tía>
  </familia>
</ListaInvitación>

Este diseño muestra cómo XML permite anidar elementos para representar relaciones jerárquicas de datos de forma clara y lógica.

XML vs JSON

Característica XML JSON
Legibilidad Buena, pero más verboso Más conciso y limpio
Uso común Servicios SOAP, configuraciones, documentos APIs REST, datos web modernos
Soporte para metadatos Sí (atributos, namespaces) No directamente
Tamaño Más pesado Más ligero
Validación DTD, XSD Schemas (JSON Schema)

Ventajas de XML

  • Altamente estructurado y flexible.
  • Soporta anotaciones y metadatos mediante atributos.
  • Estandarizado por el W3C.
  • Herramientas maduras de validación y transformación (XSLT, XPath).

Desventajas de XML

  • Verbosidad: genera archivos más grandes.
  • Lectura y escritura más lentas que JSON.
  • Complejidad innecesaria para datos simples.
  • Menos popular en APIs web modernas (donde predomina JSON).

Buenas prácticas al usar XML

  • Usa nombres de etiquetas claros y descriptivos.
  • Evita mezclar contenido en texto y atributos sin coherencia.
  • Valida tu XML con un esquema (XSD) si la estructura es crítica.
  • Declara siempre la codificación (ej. UTF-8).
  • Evita caracteres especiales sin escapar.

Tip: Aunque JSON es más común hoy en día para el intercambio de datos en la web, XML sigue siendo fundamental en entornos empresariales, sistemas legados, y estándares como SOAP, RSS, o formatos de documentos de Microsoft y OpenOffice.
