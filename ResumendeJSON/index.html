Resumen de JSON

Formato ligero para intercambio y almacenamiento de datos estructurados

¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato de texto ligero para el intercambio de datos, basado en un subconjunto del lenguaje JavaScript. Aunque deriva de JS, es lenguaje-independiente y ampliamente adoptado en aplicaciones web, móviles y backend.

Características principales

  • Ligero y fácil de leer para humanos.
  • Ampliamente soportado en APIs web y bibliotecas en casi todos los lenguajes.
  • Se representa con llaves { } para objetos y corchetes [ ] para arreglos.
  • Soporta tipos de datos: cadenas, números, booleanos, nulos, objetos y arreglos.

Usos comunes

  • Intercambio de datos entre cliente y servidor (APIs REST).
  • Archivos de configuración (.json).
  • Almacenamiento de datos en bases NoSQL (ej. MongoDB, Firebase).
  • Serialización de datos en aplicaciones móviles y de escritorio.

Reglas de sintaxis

  • Los nombres de las propiedades deben ir entre comillas dobles ("clave").
  • No se permiten comas finales ni comentarios.
  • Los valores de texto deben ir entre comillas dobles (no simples).
  • El archivo debe contener un solo valor raíz (objeto o arreglo).

Ejemplo práctico de JSON

{
  "producto": "Camiseta",
  "precio": 19.99,
  "tallas": ["S","M","L"],
  "disponible": true,
  "detalles": { "color": "azul", "marca": "Acme" }
}
Haz clic en "Probar JSON.parse()" para ver el resultado.

Ejemplo avanzado: Lista de invitados

{
  "ListaInvitación": {
    "familia": {
      "tía": [
        { "nombre": "Cristina" },
        { "nombre": "Estefanía" }
      ]
    }
  }
}

Nota: A diferencia de XML, JSON no permite múltiples elementos del mismo nombre al mismo nivel sin usar arreglos. Por eso, usamos un arreglo tía con objetos.

Ventajas de JSON

  • Sintaxis simple y mínima.
  • Parseo rápido en JavaScript (JSON.parse, JSON.stringify).
  • Menor sobrecarga de red (menos bytes).
  • Compatible con la mayoría de lenguajes modernos.

Desventajas de JSON

  • No soporta comentarios.
  • No tiene tipos de datos personalizados (ej. fechas, solo como cadenas).
  • Menos expresivo que XML para documentos complejos.
  • Estructura rígida: no permite atributos ni procesamiento declarativo (como XSLT).

Errores comunes

  • Usar comillas simples: ❌ { 'nombre': 'Ana' }
  • Coma final: ❌ { "edad": 30, }
  • Olvidar comillas en claves: ❌ { nombre: "Ana" }
  • Mezclar tipos sin estructura clara.

Cómo consumir JSON (ejemplo en JavaScript)

// Ejemplo con fetch para consumir una API que devuelve JSON
fetch('/api/producto')
  .then(resp => resp.json())
  .then(data => console.log('Producto:', data))
  .catch(err => console.error(err));

Validación y esquemas

Se puede validar la estructura de JSON con JSON Schema. No es tan estricto como XSD (XML), pero es útil para asegurarse que los datos tienen las propiedades esperadas.

Seguridad y codificación

JSON debe codificarse siempre en UTF-8. Además, al insertar JSON en HTML o JavaScript, debes evitar inyecciones (ej. escape de caracteres si se genera dinámicamente en el servidor).
